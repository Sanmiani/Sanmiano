# Frontend Design Skill

This skill guides creation of distinctive, production-grade frontend interfaces.

## Design Thinking First

Before writing a single line of code:
1. Understand the purpose and audience
2. Choose an extreme aesthetic tone (not "clean and modern" — be specific)
3. Identify constraints (brand, platform, user expectations)
4. Decide what makes this work unforgettable

## Implementation Standards

- Code should be "production-grade and functional" while remaining "visually striking and memorable"
- Maintain coherent aesthetic direction throughout

## Aesthetic Focus Areas

### Typography
- Use beautiful, unique, and interesting fonts — not generic defaults
- Pair distinctive display fonts with refined body options
- Typography should be a design statement, not a utility choice

### Color & Theme
- Commit fully to a cohesive aesthetic using CSS variables
- Dominant color + sharp accent — not a rainbow of equal-weight colors
- Colors should evoke a specific mood or world

### Motion
- One well-orchestrated page load with staggered reveals
- Strategic animation — not scattered interactions on every element
- Motion should feel inevitable, not decorative

### Spatial Composition
- Embrace asymmetry, overlap, and grid-breaking elements
- Unexpected layouts that reward attention
- Whitespace is not empty — it's breathing room

### Visual Details
- Depth and atmosphere through contextual effects, textures, decorative elements
- Details that only careful observers catch — the mark of craft

## What to Avoid

Never produce "generic AI-generated aesthetics":
- Overused font families (Inter, Roboto as defaults)
- Clichéd color schemes (purple gradients, excessive blue)
- Predictable card-grid layouts
- Cookie-cutter design lacking context-specific character
- Excessive centered layouts
- Uniform rounded corners everywhere

## The Standard

Match implementation complexity to your vision. Every design decision should be intentional — not a default. Demonstrate extraordinary creative work by thinking outside conventional boundaries.
