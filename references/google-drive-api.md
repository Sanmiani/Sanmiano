# Google Drive MCP — Reference Guide

**Status:** Connected via claude.ai MCP
**What it unlocks:** Search, read, create, and copy files in your Drive without opening it manually.

---

## What you can do

| Action | How to ask |
|---|---|
| Search files | "Find any documents about health content in my Drive" |
| Read a file | "Read my content calendar doc" |
| Create a file | "Create a Google Doc for my weekly content plan" |
| Copy a file | "Copy my post template doc for this week" |
| List recent files | "What files have I edited recently in Drive?" |
| Get file permissions | "Who has access to my content calendar?" |

---

## Best use cases for your content workflow

1. **Content calendar** — Store your weekly content plan as a Google Doc; ask Claude to populate it
2. **Research storage** — Save health stats and AI news summaries as Docs for future post ideas
3. **Caption archive** — Keep a running Doc of your best-performing captions to repurpose
4. **Brand assets** — Store logos, colour codes, and photo folders for quick reference

---

## Example prompts

```
Create a Google Doc called "Weekly Content Calendar — May 2026" 
with a table: Date | Platform | Topic | Status | Link
Fill in the next 7 days with placeholder rows.
```

```
Search my Google Drive for any health research or notes I've saved. 
Summarise what's there so I can use it for content ideas.
```

```
Create a Google Doc called "Caption Archive" with sections for 
Instagram, LinkedIn, and Facebook. Add today's date as the first entry.
```

---

## Notes

- Drive MCP reads and creates but doesn't move files between folders yet — do that manually
- Set up one folder called "AIOS Content" in Drive and store everything there for easy searching
- Google Docs created via MCP appear in your Drive root — move to your content folder after creation
